About referme.help

referme.help is a HIPAA-compliant referral management platform designed for clinical staff at physician practices, specialty groups, and health systems. It streamlines the creation, tracking, and delivery of patient referrals — including prior authorization coordination, specialist scheduling, and closed-loop response capture.

Intended users: physicians, nurses, medical assistants, referral coordinators, and administrative staff who create and manage outbound referrals on behalf of patients.

Intended setting: ambulatory care, primary care, and specialty practices operating in the United States.

SMART on FHIR Integration

referme.help supports a SMART on FHIR standalone launch (SMART App Launch Framework v2.0). Clinicians authenticate against their EHR and authorize referme.help to read a limited set of patient data in order to pre-populate a new referral without manual re-entry.

The integration is read-only by default. No data is written back to the EHR unless the user explicitly enables the optional ServiceRequest write-back feature (future roadmap).

FHIR Scopes Requested

Scope Purpose
patient/Patient.read Demographics (name, DOB, MRN) to identify and pre-populate the patient record
patient/Condition.read Active diagnoses to pre-fill referral ICD-10 codes
patient/MedicationRequest.read Current medications to populate the referral medication list
patient/AllergyIntolerance.read Documented allergies to populate the referral allergy list
openid fhirUser Identify the launching clinician for audit trail

OAuth 2.0 / PKCE Flow

referme.help uses the Authorization Code flow with PKCE (Proof Key for Code Exchange). No client secret is stored or transmitted. Tokens are encrypted at rest using AES-256 (Fernet) before storage in our database.

  • Authorization endpoint: provided by EHR SMART metadata discovery
  • Token endpoint: provided by EHR SMART metadata discovery
  • Redirect URI: https://referme.help/ehr/callback
  • SMART version: SMART App Launch Framework v2.0 (HL7)

Data Retention

Patient data imported via SMART is stored within referme.help's database under the same HIPAA-compliant data retention policy as all other patient data (configurable per organization, default 7 years). Imported fields are tagged with source imported_ehr for audit traceability.

EHR OAuth tokens are stored encrypted and are automatically revoked on user sign-out or on explicit disconnect from the EHR connection settings page.

Privacy & Security

  • All data transmitted over TLS 1.2+
  • Database hosted on Supabase (SOC 2 Type II certified)
  • OAuth tokens encrypted at rest (AES-256/Fernet)
  • Per-organization audit log for all PHI access events
  • Role-based access control (read_only → staff → clinician → coordinator → admin → owner)
  • Session revocation on sign-out; server-side session management
  • Attachment virus scanning (Cloudmersive) before storage
